# 《跨平台信使》(Cross-Platform Messenger) 游戏设计方案

## 1. 项目概述

### 1.1 基本信息
- **游戏名称**: 跨平台信使 (Cross-Platform Messenger)
- **类型**: 解谜冒险 + 悬疑叙事
- **技术栈**: Phaser.js 3.x + TypeScript + OpenClaw AI
- **平台**: Web (浏览器)
- **目标用户**: 喜爱解谜、叙事游戏的玩家

### 1.2 核心概念
玩家扮演一名网络安全调查员，通过模拟的电脑桌面界面，在多个即时通讯平台间切换调查，收集分散的线索，破解一桩神秘的网络犯罪事件。

---

## 2. 核心玩法设计

### 2.1 游戏循环
1. **探索阶段**: 在四个应用窗口间切换浏览消息
2. **收集阶段**: 识别并标记关键线索
3. **推理阶段**: 在侦探笔记本中整合线索，建立关联
4. **行动阶段**: 根据推理结果，选择对话选项推动剧情

### 2.2 界面架构
```

│  [状态栏: 时间/当前任务/线索收集进度]                         │

│  │ WhatsApp   │  │ Telegram   │  │ Discord    │  │ 邮件    ││
│  │  [窗口1]   │  │  [窗口2]   │  │  [窗口3]   │  │ [窗口4] ││
```

## 3. 四大平台设计

### 3.1 WhatsApp 窗口
- **视觉风格**: 绿色主题，气泡式对话
- **NPC角色**: 
  - "匿名线人A" - 神秘、紧张，提供关键情报
  - "受害者家属" - 焦虑、情绪化，透露背景故事
- **特殊机制**: 消息"已读"状态影响剧情分支

### 3.2 Telegram 窗口
- **视觉风格**: 蓝色主题，支持频道+私聊
- **NPC角色**:
  - "技术社群频道" - 群体讨论，技术线索
  - "黑客X" - 傲慢、挑衅，提供高难度谜题
- **特殊机制**: 加入/离开群组影响信息获取

### 3.3 Discord 窗口
- **视觉风格**: 深色主题，多频道结构
- **NPC角色**:
  - "游戏公会成员" - 轻松、日常，隐藏线索
  - "神秘用户#0000" - 只在特定时间出现
- **特殊机制**: 语音频道（文字模拟）、角色权限系统

### 3.4 邮件客户端
- **视觉风格**: 商务简洁，收件箱/发件箱/垃圾箱
- **内容类型**:
  - 官方通知、系统警告
  - 垃圾邮件中隐藏关键线索
  - 已删除邮件恢复机制
- **特殊机制**: 附件下载（图片线索）、邮件搜索

## 4. 游戏机制详解

### 4.1 线索系统

#### 线索类型
| 类型 | 说明 | 示例 |
|------|------|------|
| 人物线索 | NPC信息、关系网 | 某人的真实身份 |
| 地点线索 | 地理位置、虚拟地址 | IP地址、服务器位置 |
| 时间线索 | 时间戳、时间线 | 案发时间、不在场证明 |
| 物品线索 | 虚拟/实体物品 | 加密文件、交易记录 |
| 密码线索 | 需要解谜的内容 | 隐藏信息、暗号 |

#### 线索关联图谱
```
[线索A: 神秘IP] ────关联────► [线索B: 服务器位置] 
      │                              │
      ▼                              ▼
[线索C: 黑客身份] ◄────关联──── [线索D: 交易时间]
```

### 4.2 阅后即焚机制

#### 触发条件
- 特定NPC发送的消息（标记🔥图标）
- 倒计时 5-15 秒自动消失
- 错过无法再次查看（除非使用特殊道具）

#### 玩家应对
- 快速阅读并记忆
- 使用"截图工具"（游戏内道具，有限次数）
- 在多窗口间快速切换捕捉信息

### 4.3 时间压力系统

#### 游戏内时间
- 虚拟时间与现实时间挂钩（加速模式）
- 某些NPC只在特定时间段在线
- 事件倒计时（如：炸弹将在30分钟后引爆）

#### 时间影响
- 延迟回复可能导致线索丢失
- NPC会因等待太久而离线
- 某些剧情分支有时间窗口限制

## 5. NPC AI 设计

### 5.1 角色模板

#### 角色1: "匿名线人A"
```yaml
性格: 紧张、神秘、被害妄想
说话方式: 简短、断断续续、大量使用省略号
触发话题: 黑客攻击、内部泄密
回避话题: 个人身份、过去经历
特殊行为: 突然下线、发送自删除消息
```

#### 角色2: "黑客X"
```yaml
性格: 傲慢、挑衅、技术至上
说话方式: 技术术语、嘲讽、谜语
触发话题: 破解、漏洞、挑战
回避话题: 法律、道德、被捕风险
特殊行为: 发送加密谜题、设置陷阱信息
```

#### 角色3: "受害者家属"
```yaml
性格: 焦虑、情绪化、寻求正义
说话方式: 长篇倾诉、情感化表达
触发话题: 案件进展、家人安全
回避话题: 家人的错误、金钱问题
特殊行为: 情绪崩溃时提供关键信息
```

### 5.2 AI生成内容示例 (OpenClaw辅助)

**Prompt模板示例**:
```
你扮演一个紧张的匿名线人，正在通过WhatsApp向调查员提供
关于网络犯罪的线索。请生成3条消息：
1. 一条包含关键线索的警告消息
2. 一条因担心被追踪而突然中断的消息  
3. 一条阅后即焚的紧急消息（包含密码线索）

要求：简短、口语化、紧张感、使用中文
```

## 6. 谜题设计

### 6.1 谜题类型

#### 类型A: 跨平台信息拼图
**描述**: 线索分散在多个平台，需要整合
**示例**: 
- WhatsApp获得一半IP地址
- Discord获得另一半IP地址
- 组合后在邮件系统追踪

#### 类型B: 时间线重构
**描述**: 按正确顺序排列事件
**示例**:
- 从不同平台收集带时间戳的消息
- 在侦探笔记本中拖拽排序
- 揭示事件真相

#### 类型C: 密码破解
**描述**: 解密隐藏信息
**示例**:
- Telegram中的加密消息
- Discord中的图片隐写
- 需要找到密钥才能解密

#### 类型D: 身份推理
**描述**: 通过对话推断NPC真实身份
**示例**:
- 分析四个平台的语言习惯
- 对比时间在线规律
- 揭穿伪装的账号

### 6.2 难度曲线
| 章节 | 谜题复杂度 | 时间压力 | 平台数量 |
|------|-----------|---------|---------|
| 第一章 | 单一平台 | 无 | 1-2个 |
| 第二章 | 双平台关联 | 轻度 | 2-3个 |
| 第三章 | 多平台整合 | 中度 | 3-4个 |
| 终章 | 全平台+时间压力 | 重度 | 4个+限时 |

### 7.2 核心类设计

#### ClueSystem (线索系统)
```typescript
class ClueSystem {
  private clues: Map<string, Clue>;
  private discoveredClues: Set<string>;
  private connections: Map<string, string[]>;
  
  discoverClue(clueId: string): void;
  connectClues(clueA: string, clueB: string): boolean;
  checkPuzzleSolved(puzzleId: string): boolean;
  getClueHints(): string[];
}
```

#### NPCDialogueSystem (NPC对话系统)
```typescript
class NPCDialogueSystem {
  private npcs: Map<string, NPC>;
  private dialogueHistory: Message[];
  
  generateResponse(npcId: string, playerInput: string): Promise<string>;
  addSelfDestructMessage(content: string, duration: number): void;
  triggerEvent(eventId: string): void;
}
```

#### TimePressureSystem (时间压力系统)
```typescript
class TimePressureSystem {
  private gameTime: number;
  private countdowns: Map<string, number>;
  
  startCountdown(id: string, seconds: number, callback: Function): void;
  pauseTime(): void;
  getCurrentGameTime(): Date;
  isNPCOnline(npcId: string): boolean;
}
```

### 7.3 OpenClaw AI 集成点

| 功能 | AI用途 | 实现方式 |
|------|--------|---------|
| NPC对话生成 | 根据上下文生成自然回复 | API调用，预设角色prompt |
| 线索文本创作 | 生成谜题描述、隐藏信息 | 批量生成后人工筛选 |
| 谜题逻辑设计 | 设计线索关联、难度平衡 | 辅助设计，人工调整 |
| 剧情分支 | 生成不同选择的后果 | 大纲设计，AI扩展 |

## 8. 美术风格

### 8.1 视觉方向
- **整体风格**: 扁平化UI + 微立体效果
- **配色方案**: 
  - WhatsApp: #25D366 绿
  - Telegram: #0088CC 蓝
  - Discord: #5865F2 紫
  - 邮件: #EA4335 红
- **字体**: Inter (正文), JetBrains Mono (代码/密码)

### 8.2 UI元素
- 窗口边框：细微阴影 + 圆角
- 消息气泡：平台特色配色
- 线索标记：金色高亮 + 脉冲动画
- 阅后即焚：火焰图标 + 倒计时条

## 9. 音效与音乐

### 9.1 音效设计
| 事件 | 音效风格 |
|------|---------|
| 新消息 | 各平台特色提示音（变奏版） |
| 发现线索 | 清脆的"叮"声 + 低频脉冲 |
| 阅后即焚 | 纸张燃烧声 + 倒计时滴答 |
| 谜题解开 | 解锁声 + 上升旋律 |
| NPC上线 | 轻微登录提示 |

### 9.2 背景音乐
- **探索阶段**: 低沉的电子氛围音乐
- **紧张阶段**: 加速的节奏 + 心跳声
- **解谜阶段**: 极简的合成器音色
- **剧情高潮**: 弦乐 + 电子混合
