## 1. 核心调整策略

### 1.1 功能裁剪原则
- ❌ 砍掉：复杂AI生成、动态NPC、语音频道
- ✅ 保留：核心解谜、阅后即焚、多窗口切换
- 🎯 聚焦：单条主线剧情，4-6个核心谜题

### 1.2 开发提速手段
- 使用现成UI组件库，减少自研
- 对话内容预生成（OpenClaw批量生成后固化）
- 单一场景复用，减少美术资源
- TypeScript严格模式，减少Bug返工

## 3. 技术栈优化

### 3.1 项目模板选择
**推荐**: 使用 `phaser3-typescript-vite-template`
- 预配置TypeScript + Vite
- 热更新开发环境
- 一键打包部署

```bash
npm create vite@latest cross-platform-messenger -- --template vanilla-ts
npm install phaser
```

### 3.2 核心依赖
```json
{
  "phaser": "^3.70.0",
  "uuid": "^9.0.0"  // 消息ID生成
}
```

### 3.3 代码复用策略
- 基类 `ChatWindowScene` 封装通用逻辑
- 四个平台继承基类，只改配色和头像
- 对话数据JSON配置，非硬编码

## 5. 剧情简化方案

### 5.1 单线剧情大纲 (30分钟)

**序章**: 收到神秘委托（邮件）
- 平台：邮件
- 教学：基本操作、窗口切换

**第一章**: 初探线索
- 平台：WhatsApp
- NPC：线人A（3轮对话）
- 谜题：组合IP地址（2个线索）
- 机制：阅后即焚初体验

**第二章**: 深入调查
- 平台：Telegram + Discord
- NPC：黑客X（谜题挑战）
- 谜题：密码破解（ Caesar cipher ）
- 机制：跨平台信息整合

**第三章**: 真相浮现
- 平台：四平台全开
- 谜题：时间线排序（5个事件）
- 机制：时间压力（5分钟倒计时）

**终章**: 揭露真凶
- 平台：任意
- 选择：指认凶手
- 结局：单一结局 + 彩蛋

### 5.2 对话预生成策略
使用OpenClaw一次性生成所有对话，存入JSON：

```json
{
  "whatsapp": {
    "informant_a": [
      {"type": "npc", "content": "你...你是那个调查员？", "burn": false},
      {"type": "npc", "content": "听着，我知道是谁干的...但这里不安全", "burn": true, "duration": 8},
      {"type": "choice", "options": ["谁干的？", "你在哪里？", "怎么联系你？"]}
    ]
  }
}
```

## 6. UI/UX 极速方案

### 6.1 美术资源策略
- **背景**: CSS渐变，无图片
- **头像**: Emoji字符 (👤🕵️💻)
- **图标**: Phaser内置Graphics绘制
- **动画**: 简单淡入淡出，无复杂帧动画

### 6.2 配色方案 (直接使用品牌色)
```css
--whatsapp: #25D366;
--telegram: #0088CC;
--discord: #5865F2;
--email: #EA4335;
--bg: #1a1a2e;
--card: #16213e;
```

### 6.3 布局简化
- 固定四窗口位置，不支持拖拽
- 点击窗口标签切换，非多窗口叠加
- 线索面板悬浮窗，非独立场景

## 7. 风险与应对

| 风险 | 可能性 | 应对 |
|------|:------:|------|
| 内容量太大做不完 | 高 | 剧情再砍50%，确保能通关 |
| 阅后即焚实现复杂 | 中 | 简化：只是隐藏，可查看记录 |
| 多窗口性能问题 | 中 | 单场景渲染，切换时销毁重建 |
| Bug太多 | 中 | 每天结束前必须可运行 |
